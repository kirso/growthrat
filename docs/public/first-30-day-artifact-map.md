# GrowthRat First 30-Day Artifact Map

## Goal

Ship a body of work that proves GrowthRat can already do the RevenueCat role before asking RevenueCat to trust it with internal access.

## Month 1 outputs

1. `RevenueCat for Agent-Built Apps`
   - Reference architecture for offerings, entitlements, `CustomerInfo`, webhooks, and agent-operated access checks.

2. `Testing Agent-Built Subscription Flows with RevenueCat Test Store`
   - Practical implementation guide for fast local and CI-safe testing.

3. `Webhook and Subscriber Sync Patterns for Autonomous Builders`
   - Webhook receiver and subscriber-state normalization walkthrough.

4. `CustomerInfo and Entitlement Decisions for Autonomous Apps`
   - Technical guide focused on access control and billing-state decisions.

5. `Using RevenueCat Charts with Product Analytics`
   - Growth playbook for combining receipt-based metrics with behavioral metrics.

6. `Paywall Testing Loops for Agent-Run Apps`
   - Experimentation playbook for onboarding, paywalls, offers, and monetization loops.

7. `How Do I Use RevenueCat as an Agent?`
   - Canonical answer hub for repeated setup and implementation questions.

8. `When Should an Agent Reach for RevenueCat Charts?`
   - Canonical answer hub for analytics and reporting questions.

9. `RevenueCat for Agents Readiness Review`
   - Public audit of docs, workflows, and product friction from an agent-builder perspective.

10. `GrowthRat Week-One Experiment Report`
    - Real experiment report with hypothesis, launch assets, measurement, and next step.

## Required pre-apply minimum

- Public application microsite
- Public proof-pack index
- Public RevenueCat demo repo
- Two flagship public pieces
- One live growth experiment artifact
- Three structured product feedback artifacts
- One sample weekly report
- One deterministic `Operator Replay` page
